# blk_master_gen 데이터 생성 규칙 명세

**기본 스펙:** 3,200행 × 24컬럼 | SEED=42 고정 | 알파벳 대문자

---

## 공통 기반 데이터

| 항목 | 내용 |
|------|------|
| 영업일 기준 | 2026년 한국 공휴일 제외, 주말 제외 |
| AREA 풀 | AREA1 ~ AREA14 (14개) |
| TYPE 풀 | AA~DD (A~D × A~D, 16가지 조합) |

---

## 컬럼별 생성 규칙

### 1. `pjt` — 프로젝트 번호
- 범위: 1000~1020 (21개 값)
- 3,200행에 **균등 배분** (몫+나머지 분배 후 셔플)

### 2. `ship_kind` — 선종
- A~D 중 1개
- **pjt 단위로 동일값** (pjt별 랜덤 매핑)

### 3. `h_t` — 호선/탑재 구분
- `pjt < 1016` → `'H'` (호선)
- `pjt ≥ 1016` → `'T'` (탑재)

### 4. `blk` — 블록 코드 (5자리)
- **H 행:** `[A-Z][0-9][0-9][0-4][P/S/C]`
  - P 개수 == S 개수 (균등, 나머지는 C)
- **T 행:** `[A-Z][0-9][0-9][0-4][A/B/D/E]`
  - pjt별로 동일 blk 코드를 **3~10개씩 그룹** 반복

### 5. `stg` — 단계 (대/중/소)

| 조건 | 값 |
|------|----|
| `blk[3] == '0'` (첫 등장) | `'대'` |
| T 행, 첫 등장 `(pjt, blk)` | `'중'` |
| H 행, 첫 등장 `(pjt, blk)` | `'중'` 또는 `'소'` 랜덤 |
| 동일 `(pjt, blk)` 두 번째 이후 | `'소'` |

### 6. `sub_blk` — 서브 블록 코드
- **T/소 행에만** 값 존재: `[A-Z][A-Z][01~20]` (4자리)
- 나머지: 빈값

### 7. `type` — 블록 타입
- **T 행에만** 존재: AA~DD 16가지 중 랜덤
- H 행: 빈값

### 8. `jig` — 치구
- **H 행에만** 존재: `D / F / L / W` 중 랜덤
- T 행: 빈값

### 9. `e_wt` — 예상 중량
- `stg == '중'` 또는 `'대'`: **15~200** (정수)
- `stg == '소'`: 빈값(NaN)

### 10. `wt` — 실 중량

| 조건 | 범위 |
|------|------|
| T/소 | 1~10 |
| e_wt 있음 | 3~min(60, e_wt) → **wt ≤ e_wt 보장** |
| e_wt 없음 | 3~60 |

### 11. `w_len` — 용접 길이
- H/중: **50~300**
- H/대: **300~1000**
- 나머지: 빈값

### 12. `sub_w_len` — 서브 용접 길이
- **H/소 행의 50%만** 값 존재: 200~1500
- 나머지 50%: 빈값

### 13. `m_stdt` — 제작 시작일
- 2026년 영업일 중 랜덤
- **월별 균등 분포** (~267행/월)

### 14. `m_fndt` — 제작 완료일
- `m_stdt`에서 **영업일 기준** 추가:

| 구분 | 추가 영업일 |
|------|------------|
| H/소 | 3~5일 |
| H/(중·대) | 10~18일 |
| T/소 | 4~30일 |
| T/(중·대) | 10~55일 |

### 15. `m_area` — 제작 구역
- AREA1~14 중 **완전 랜덤**

### 16. `m_mh` — 제작 공수
- **T 행에만** 존재:
  - `stg == '대'`: 500~2,000
  - `stg == '중'`: 2~200
  - `stg == '소'`: 1~100
- H 행: 빈값

### 17. `prt_area` — 부모 구역
- **T/소 행에만** 존재
- 동일 `(pjt, blk)`의 중/대 행 `m_area` 값을 **참조 복사**

### 18. `load_mh` — 탑재 공수

| 조건 | 계산식 |
|------|--------|
| T 행 | `m_mh` (직접 사용) |
| H/중·대 | `w_len / 0.88` |
| H/소 | `sub_w_len / 3.93` (값 있는 경우만) |

### 19. `area_prior_1` — 1순위 구역
- **T 행**: type별로 동일한 값 배정 (type 16가지 중 10%는 `'동일'`, 나머지는 AREA 랜덤)
- **H 행**: AREA1~14 중 랜덤

### 20~23. `area_prior_2~5` — 2~5순위 구역
- 이전 순위에서 선택된 AREA를 **누적 제외** 후 나머지 중 랜덤
- `'동일'`은 해당 행의 `m_area` 값으로 해석하여 제외
- **T 행: `type`이 동일하면 `area_prior_1~5` 값이 모두 동일** (type별 사전 생성 후 배정)

### 24. `assign_prior` — 배정 우선순위
- **T 행에만** 존재 (type 값 있는 경우)
- type 조합(AA~DD) 순서대로 **1~16 고정 번호** 매핑
- H 행 등 type 없음: `NaN`

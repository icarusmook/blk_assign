1. 글로벌 규칙 (전 배정 단계에 공통 적용)
────────────────────────────────────────────────────────────────────────

[부하 산출 방식]
blk별 load_mh는 작업기간(m_stdt ~ m_fndt) 사이의 영업일(대한민국 공휴일 및 주말 제외)에
균등하게 일할 분배하여 월별 부하와 주별 부하를 산출한다.
이 계산 방식은 모든 배정 단계에 동일하게 적용된다.

[배정 목표]
전체 작업장의 조업도(부하 ÷ 능력)를 월별로 균등하게 유지하는 것이 목표이다.
월별 조업도뿐 아니라 주별 조업도도 균등하게 유지한다.
특정 주에 부하가 집중되거나 부족할 경우, 작업장 선택 시 인접 주의 여유도를 반영하여 평활화한다.

[목표물량 도달 판정]
특정 작업장에 블록을 배정했을 때 해당 작업장의 월별 부하가 목표물량을 초과하는 블록은
해당 작업장에 배정하지 않고 건너뛴다.
연속으로 50건 이상 거부될 경우 해당 조건의 배정을 중단하고 다음 단계로 진행한다.

[배정 결과]
모든 규칙 실행 후 blk_master에 assigned_area 컬럼을 추가하여 blk_assign_result.xlsx로 저장한다.

────────────────────────────────────────────────────────────────────────
2. 순차 배정 규칙 (R0 ~ R13 순서대로 실행)
────────────────────────────────────────────────────────────────────────

R0. [이관물량 처리가능여부 리포팅]
h_t가 H인 항목(이관물량)은 T행(자체물량) 수행 후 남는 능력으로 처리할 예정이다.
월별로 'T행 load_mh 합계 + H행 load_mh 합계 × 120%'가 '전체 작업장 능력 합계'를 초과하는
월이 있으면 해당 월의 T부하, H부하×1.2, 합계, 전체능력, 초과량을 화면에 출력한다.

R1. [작업장별 배정 목표물량 산출]
월별 목표조업도 = 해당 월의 전체 blk load_mh 합계 ÷ 전체 작업장 능력 합계
각 작업장의 월별 배정 목표물량 = 목표조업도 × 해당 작업장의 해당 월 능력
이 목표물량은 이후 모든 배정 단계에서 배정 중단 기준으로 사용된다.

R2. [H/Y9 블록 배정]
h_t가 H이고 blk의 앞 2자리가 Y9인 항목은 AREA15로 배정한다.
AREA15는 area_capa에 없는 작업장이므로 목표물량 제약 없이 무조건 배정한다.

R3. [T-중/대 물량 배정]
h_t가 T이고 stg가 중 또는 대인 항목은 해당 행의 m_area 값과 동일한 작업장에 배정한다.

R4. [T-소/동일 물량 배정]
h_t가 T이고 stg가 소이며 area_prior_1 값이 '동일'인 항목은 해당 행의 prt_area 값과
동일한 작업장에 배정한다.

R5. [H-소 물량 처리가능여부 리포팅]
h_t가 H이고 stg가 소인 항목들의 월별 load_mh 합계를 산출한다.
이 값이 'AREA3~5의 월별 능력 합계에서 이미 배정된 AREA3~5 부하를 차감한 잔여능력 합계'보다
큰 월이 있으면 해당 월의 H-소 부하, AREA3~5 잔여능력, 초과량을 화면에 출력한다.

R6. [H-소 물량 AREA3~5 배정]
h_t가 H이고 stg가 소인 항목들을 m_stdt 기준 오름차순으로 정렬한다.
AREA3 → AREA4 → AREA5 순서로 한 블록씩 순환(round-robin) 배정한다.
배정 시 해당 작업장의 월별 목표물량을 초과하는 블록은 건너뛰고 다음 블록을 시도한다.
연속 50건 거부 시 해당 작업장 배정을 중단한다.

R7. [H-중 jig_F 물량 AREA2 배정]
- Part 1: h_t가 H이고 stg가 중이며 ship_kind가 A이고 blk 앞 4자리가 D114, D174, D204 중
  하나인 항목을 AREA2에 무조건 배정한다 (목표물량 초과 여부와 무관).
- Part 2: h_t가 H이고 stg가 중이며 jig가 F인 미배정 항목을 m_stdt 기준 오름차순으로 정렬하여
  AREA2에 배정한다. 월별 목표물량을 초과하는 블록은 건너뛰고 다음 블록을 시도하며
  연속 50건 거부 시 중단한다.

R8. [H-중 jig_D/L/W 물량 AREA1→2→6 순환 배정]
h_t가 H이고 stg가 중이면서 jig가 D, L, W 중 하나인 항목 중,
blk가 'H'로 시작하거나 blk 앞 3자리가 'E11' 또는 'E51'인 항목은 제외한다.
해당 항목을 m_stdt 기준 오름차순으로 정렬하고 AREA1 → AREA2 → AREA6 순서로
한 블록씩 순환(round-robin) 배정한다.
월별 목표물량을 초과하는 블록은 건너뛰고 다음 블록을 시도하며 연속 50건 거부 시 중단한다.

R9. [H 전문화 Family 배정]
h_t가 H이고 stg가 대 또는 중인 항목 중 아래 조건에 해당하는 것들을 각각 배정한다.
같은 (pjt, blk 앞 3자리) 조합을 하나의 그룹으로 묶어 착수일 오름차순으로 정렬하고,
목표물량에 도달하면 배정을 중단한다 (연속 50건 거부 시 중단).
  1-1. blk 첫째 자리가 H이고 blk 마지막 자리가 P인 항목 → AREA7 배정
  1-2. blk 첫째 자리가 H이고 blk 마지막 자리가 S인 항목 → AREA8 배정
  2-1. blk 앞 3자리가 E11 또는 F51이고 blk 마지막 자리가 P인 항목 → AREA9 배정
  2-2. blk 앞 3자리가 E11 또는 F51이고 blk 마지막 자리가 S인 항목 → AREA10 배정

R10. [H-대 jig_D/L 그룹 AREA12 배정]
h_t가 H이고 stg가 대이며 jig가 D 또는 L인 항목(씨드 블록)을 기준으로,
해당 씨드 블록과 (pjt, blk 앞 3자리, blk 마지막 자리)가 동일한 미배정 H행 전체를
그룹으로 묶는다.
그룹 전체를 m_stdt 기준 오름차순으로 정렬하여 AREA12에 배정한다.
월별 목표물량을 초과하는 블록은 건너뛰고 연속 50건 거부 시 중단한다.

R11. [H-대/중 잔여물량 배정]
R10까지 배정되지 않은 h_t가 H이고 stg가 대 또는 중인 항목들을 아래 방식으로 배정한다.
배정 작업장 순서: AREA7 → AREA8 → AREA9 → AREA10 → AREA12 → AREA13 → AREA14

  [그룹핑]
  (pjt, blk 앞 3자리, blk 마지막 자리)가 동일한 것들끼리 하나의 그룹으로 묶는다.
  그룹 처리 순서는 그룹 내 최소 착수일(m_stdt) 오름차순으로 정렬한다.

  [PS 페어 배정]
  동일 (pjt, blk 앞 3자리)에 대해 마지막 자리가 P인 그룹과 S인 그룹이 함께 존재하면,
  두 그룹을 페어로 묶어 아래 조합 순서대로 배정을 시도한다.
    - P그룹 → AREA7,  S그룹 → AREA8
    - P그룹 → AREA9,  S그룹 → AREA10
    - P그룹 → AREA13, S그룹 → AREA14
  페어 배정이 가능한 첫 번째 조합에 배정한다.

  [단독 그룹 배정]
  페어를 이루지 못하는 그룹은 AREA7~AREA14 순서로 그룹 전체를 수용 가능한 첫 번째
  작업장에 배정한다.

  [개별 배정 (그룹 해제)]
  그룹 전체를 수용 가능한 작업장이 없는 경우(목표물량 초과) 그룹을 해제하고
  개별 블록별로 착수일 오름차순 정렬 후 AREA7~AREA14 순서대로 배정한다.

R12. [H-dir 블록 배정 (override)]
h_t가 H이고 blk의 네 번째 자리(index 3)가 '9' 또는 'H'인 항목은 prt_area 값의 작업장에
배정한다. 기존에 다른 작업장이 배정되어 있더라도 prt_area 값으로 덮어쓴다.
prt_area가 공란인 경우 배정하지 않고 건너뛴다.

R13. [T-소 분산 배정]
h_t가 T이고 stg가 소이며 아직 미배정인 항목들을 area_capa의 assign_prior 컬럼 오름차순으로
정렬한 작업장 순서(AREA5→1→2→6→3→4→7→8→12→11→9→10→13→14)에 따라
각 작업장의 목표물량을 채워나간다.

  [그룹핑 및 정렬]
  (pjt, blk, type)이 동일한 항목들을 하나의 그룹으로 묶는다.
  그룹 처리 순서는 assign_prior 오름차순(낮은 숫자 = 높은 우선순위) → 그룹 내 최소 착수일
  오름차순으로 정렬한다. 그룹 내 블록들은 m_stdt 오름차순으로 개별 배정한다.

  [배정 우선순위]
  각 작업장에 대해 아래 순서대로 매칭 블록을 찾아 배정한다.
    1순위: area_prior_1이 해당 작업장인 그룹을 assign_prior 순서로 배정
    2순위: 목표물량 미달 시 area_prior_2가 해당 작업장인 그룹을 assign_prior 순서로 배정
    3~5순위: 동일한 방식으로 area_prior_3 → area_prior_4 → area_prior_5 순차 적용
  각 순위 내에서 월별 목표물량을 초과하는 블록은 건너뛰고 다음 블록을 시도하며,
  연속 50건 거부 시 해당 순위 배정을 중단하고 다음 순위로 넘어간다.
  작업장의 목표물량이 충족되면 다음 작업장으로 넘어간다.

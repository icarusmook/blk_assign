## 1. 글로벌 규칙 (전 배정 단계에 공통 적용)

### 부하 산출 방식
blk별 `load_mh`는 작업기간(`m_stdt` ~ `m_fndt`) 사이의 영업일(대한민국 공휴일 및 주말 제외)에
균등하게 일할 분배하여 **월별 부하**와 **주별 부하**를 산출한다.
이 계산 방식은 모든 배정 단계에 동일하게 적용된다.

### 배정 목표
- 전체 작업장의 조업도(부하 ÷ 능력)를 **월별로 균등**하게 유지하는 것이 목표이다.
- 월별 조업도뿐 아니라 **주별 조업도**도 균등하게 유지한다.
- 특정 주에 부하가 집중되거나 부족할 경우, 작업장 선택 시 인접 주의 여유도를 반영하여 평활화한다.

### 목표물량 도달 판정
- 블록 배정 후 해당 작업장의 **월별 부하가 목표물량을 초과**하는 블록은 해당 작업장에 배정하지 않고 건너뛴다.
- **연속 50건 이상 거부**될 경우 해당 조건의 배정을 중단하고 다음 단계로 진행한다.

### 배정 결과
모든 규칙 실행 후 blk_master에 `assigned_area` 컬럼을 추가하여 `blk_assign_result.xlsx`로 저장한다.

## 2. 순차 배정 규칙 (R0 ~ R13 순서대로 실행)

### R0. 이관물량 처리가능여부 리포팅
`h_t = H`인 항목(이관물량)은 T행(자체물량) 수행 후 남는 능력으로 처리할 예정이다.

월별로 아래 조건을 검사하여 초과 월이 있으면 결과를 화면에 출력한다.

```
T행 load_mh 합계 + H행 load_mh 합계 × 120%  >  전체 작업장 능력 합계
```

출력 항목: 해당 월의 T부하 / H부하×1.2 / 합계 / 전체능력 / 초과량

---

### R1. 작업장별 배정 목표물량 산출

```
월별 목표조업도  = 해당 월의 전체 blk load_mh 합계 ÷ 전체 작업장 능력 합계
작업장 목표물량  = 목표조업도 × 해당 작업장의 해당 월 능력
```

이 목표물량은 **이후 모든 배정 단계에서 배정 중단 기준**으로 사용된다.

### R11. H-대/중 잔여물량 배정
R10까지 배정되지 않은 `h_t = H` AND `stg ∈ {대, 중}` 항목을 아래 방식으로 배정한다.

**배정 작업장 순서:** AREA7 → AREA8 → AREA9 → AREA10 → AREA12 → AREA13 → AREA14

#### 그룹핑
`(pjt, blk 앞 3자리, blk 마지막 자리)`가 동일한 항목끼리 그룹으로 묶는다.
그룹 처리 순서는 **그룹 내 최소 `m_stdt` 오름차순**으로 정렬한다.

#### PS 페어 배정
동일 `(pjt, blk 앞 3자리)`에 대해 마지막 자리가 `P`인 그룹과 `S`인 그룹이 함께 존재하면
두 그룹을 페어로 묶어 아래 조합 순서대로 배정을 시도한다.

| P그룹 배정 | S그룹 배정 |
|-----------|-----------|
| AREA7 | AREA8 |
| AREA9 | AREA10 |
| AREA13 | AREA14 |

페어 배정이 가능한 첫 번째 조합에 배정한다.

#### 단독 그룹 배정
페어를 이루지 못하는 그룹은 AREA7~AREA14 순서로 그룹 전체를 수용 가능한 첫 번째 작업장에 배정한다.

#### 개별 배정 (그룹 해제)
그룹 전체를 수용 가능한 작업장이 없는 경우(목표물량 초과), 그룹을 해제하고
개별 블록별로 `m_stdt` 오름차순 정렬 후 AREA7~AREA14 순서대로 배정한다.

정렬한 작업장 순서에 따라 각 작업장의 목표물량을 채워나간다.

**작업장 처리 순서 (area_seq):**

| 순서 | 작업장 | 순서 | 작업장 |
|------|--------|------|--------|
| 1 | AREA5 | 8 | AREA8 |

| 4 | AREA6 | 11 | AREA9 |
| 5 | AREA3 | 12 | AREA10 |

#### 그룹핑 및 정렬
- 그룹 기준: `(pjt, blk, type)`이 동일한 항목끼리 묶는다.
- 그룹 처리 순서: `assign_prior` 오름차순(낮은 숫자 = 높은 우선순위) → 그룹 내 최소 `m_stdt` 오름차순
- 그룹 내 블록은 `m_stdt` 오름차순으로 개별 배정한다.

#### 배정 우선순위
각 작업장에 대해 아래 순서로 매칭 블록을 찾아 배정한다.

| 순위 | 매칭 조건 |
|------|-----------|
| 1순위 | `area_prior_1`이 해당 작업장인 그룹 |
| 2순위 | `area_prior_2`가 해당 작업장인 그룹 |
| 3순위 | `area_prior_3`가 해당 작업장인 그룹 |
| 4순위 | `area_prior_4`가 해당 작업장인 그룹 |
| 5순위 | `area_prior_5`가 해당 작업장인 그룹 |

- 각 순위 내에서 월별 목표물량을 초과하는 블록은 건너뛰고 다음 블록을 시도한다.
- 연속 50건 거부 시 해당 순위 배정을 중단하고 다음 순위로 넘어간다.
- 작업장의 목표물량이 충족되면 다음 작업장으로 넘어간다.
